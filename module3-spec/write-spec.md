# Spec 작성해보기

## Spec 시작하기

Kiro에서 Spec은 채팅이 아니라 **전용 패널**에서 작성합니다.

1. Kiro 좌측 사이드바에서 **Specs** 아이콘을 클릭합니다
2. **New Spec** 버튼을 클릭합니다
3. Spec 이름을 입력합니다: `사고 대응 보고서 생성기`

***

## Step 1: 요구사항 작성하기

Spec 패널이 열리면, 아래 내용을 프롬프트에 입력합니다.

**📋 Spec 프롬프트에 입력**

```
편의점에서 사고가 발생했을 때 대응 보고서를 자동으로 생성해주는 웹앱을 만들고 싶어.

입력 항목:
- 사고 유형 선택 (화재, 도난, 고객 부상, 식품 안전, 시설 파손)
- 발생 일시
- 발생 장소 (매장 내 위치)
- 상황 설명 (자유 텍스트)
- 긴급도 (상/중/하)

출력:
- 사고 보고서 초안 (공식 양식)
- 긴급도에 따른 보고 라인 안내
- 즉시 조치 사항 체크리스트
- 보고서 복사 버튼
```

Enter를 누르면 Kiro가 **Requirements(요구사항)** 문서를 자동 생성합니다.

***

## Step 2: Requirements 확인하기

Kiro가 생성한 Requirements 문서를 살펴보세요.

여러분이 간단히 적은 내용을 AI가 다음과 같이 **체계적으로 정리**해줍니다:

* 기능 요구사항 (어떤 기능이 있어야 하는지)
* 화면 구성 (어떤 페이지가 필요한지)
* 데이터 흐름 (입력 -> 처리 -> 출력)
* 예외 상황 처리

> **ℹ️ 참고**
> **여러분이 한 것**: 아이디어를 자연어로 적음\
> **Kiro가 한 것**: 그것을 개발 가능한 요구사항 문서로 정리

***

## Step 3: Design 문서 생성하기

Requirements가 만족스러우면, **Design** 단계로 넘어갑니다.

Kiro가 Requirements를 바탕으로 **설계 문서**를 자동 생성합니다:

* 화면 레이아웃 구성 (어떤 화면이 어디에 배치되는지)
* 화면 간 연결 구조 (어떤 버튼을 누르면 어디로 가는지)
* 데이터 흐름 (입력한 정보가 어떻게 처리되는지)

> **✅ 완료**
> **여기까지가 오늘의 Spec 체험입니다!**\
> \
> 정리하면:\
> \
> 1. 여러분이 아이디어를 자연어로 적으면\
> 2. Kiro가 **요구사항(Requirements)** 을 정리해주고\
> 3. 그 다음 **설계도(Design)** 까지 자동으로 그려줍니다\
> \
> 실제 프로젝트에서는 이 설계도를 바탕으로 코드를 자동 생성하는 단계까지 진행할 수 있습니다.

***

## 바이브 코딩과 비교

| | 바이브 코딩 (Module 2) | Spec (Module 3) |
| --- | --- | --- |
| **과정** | 채팅으로 바로 코드 생성 | 요구사항 → 설계 → (코드 생성) |
| **느낌** | "일단 만들어봐!" | "계획부터 세우자" |
| **언제 좋을까** | 빠르게 프로토타입 만들 때 | 제대로 된 앱을 만들 때 |

두 방식을 **섞어 쓸 수도 있습니다.**\
예: Spec으로 큰 틀을 잡고, 세부 조정은 바이브 코딩으로.
