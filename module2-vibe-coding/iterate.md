# 대화로 다듬기

## 앱은 만들어졌지만, 아직 완벽하지 않죠?

첫 번째 프롬프트로 기본 틀을 만들었습니다.\
이제 **대화를 주고받으면서** 앱을 점점 더 좋게 만들어보겠습니다.

> **ℹ️ 참고**
> 이 과정이 바이브 코딩의 **핵심**입니다.\
> 한 번에 완벽한 앱을 만들려고 하지 마세요.\
> 여러 번 요청하며 다듬어가는 것이 자연스러운 방법입니다.

***

## 실습 1: 디자인 다듬기

아래 프롬프트를 하나씩 입력해보세요. 매번 결과를 브라우저에서 확인합니다.

### 1-1. 배경 수정

**📋 Kiro Chat에 입력**

```
전체 배경을 밝은 회색으로 바꾸고, 채팅 영역은 흰색 카드 형태로 만들어줘.
카드에 약간의 그림자 효과를 넣어줘.
```

브라우저에서 변화를 확인하세요. 좀 더 깔끔해졌나요?

### 1-2. 헤더 개선

**📋 Kiro Chat에 입력**

```
상단 헤더를 GS25 파란색 배경에 흰색 글자로 바꿔줘.
"GS25 규정 도우미" 제목 아래에 "470페이지 매뉴얼, 이제 물어보세요" 부제목도 추가해줘.
```

### 1-3. 예시 버튼 스타일

**📋 Kiro Chat에 입력**

```
자주 묻는 질문 예시 버튼을 둥근 알약 모양으로 바꾸고,
파란색 테두리에 흰색 배경으로 만들어줘.
마우스를 올리면 파란색 배경으로 바뀌게 해줘.
```

## 실습 3: Steering 효과 체험하기

이제 Steering을 바꿔서 **같은 앱의 분위기가 바뀌는 것**을 확인해보겠습니다.

### 3-1. Steering 수정

좌측 파일 탐색기에서 `.kiro/steering.md`를 열고, **AI 답변 규칙** 부분을 아래로 바꿔보세요:

**📋 steering.md의 AI 답변 규칙을 이렇게 수정**

```markdown
## AI 답변 규칙
- 친근한 반말 사용 ("~해!", "~야!")
- 답변에 이모지를 적극 활용
- 최대한 상세하고 친절하게 설명
- 중요한 부분은 볼드로 강조
```

저장한 후 (`Ctrl+S`), Kiro Chat에서:

**📋 Kiro Chat에 입력**

```
steering 규칙이 바뀌었어. 앱의 AI 답변 스타일을 steering에 맞게 업데이트해줘.
```

### 3-2. 비교해보기

같은 질문("반품 절차가 어떻게 되나요?")을 했을 때 답변 톤이 어떻게 달라졌나요?

| Before (존댓말 Steering) | After (반말 Steering) |
| --- | --- |
| "반품 절차는 다음과 같습니다..." | "반품은 이렇게 하면 돼!" |
| 딱딱하고 공식적 | 친근하고 캐주얼 |

> **✅ 완료**
> **Steering 하나만 바꿨을 뿐인데 앱의 성격이 바뀌었습니다!**\
> 이것이 Steering의 힘입니다. 코드를 하나도 건드리지 않고 AI의 행동을 제어할 수 있어요.
