# Module 2: 바이브 코딩 - 말로 앱 만들기

## 이번 Module에서 할 것

| 순서 | 내용 | 시간 |
| --- | --- | --- |
| 1 | 첫 번째 프롬프트로 앱 생성 | 15분 |
| 2 | 대화로 다듬기 (디자인, 기능 수정) | 20분 |
| 3 | @파일로 규정 데이터 연결하기 | 15분 |

***

## 바이브 코딩이란?

**바이브 코딩(Vibe Coding)** = 코드를 직접 쓰지 않고, **자연어 대화**로 AI에게 앱을 만들게 하는 방식

| 기존 코딩 | 바이브 코딩 |
| --- | --- |
| 개발자가 코드를 한 줄씩 작성 | 한글로 "이런 거 만들어줘" 요청 |
| 버그를 직접 찾아서 수정 | "이 부분 이상해, 고쳐줘" 대화 |
| 디자인을 CSS로 조정 | "좀 더 깔끔하게 바꿔줘" 요청 |

{% hint style="info" %}
**MISO에서 프롬프트 작성하셨던 경험**, 그대로 활용하시면 됩니다.\
다만 MISO에서는 에이전트 로직을 만들었다면, 여기서는 **눈에 보이는 앱**을 만듭니다.
{% endhint %}

## 프롬프트 주문서

막힐 때 참고하세요!

| 유형 | 예시 프롬프트 |
| --- | --- |
| **기본 주문** | "~하는 웹페이지 만들어줘" |

| **톤 변경** | "답변을 좀 더 친근하게 바꿔줘" |
| **레이아웃** | "모바일에서도 잘 보이게 해줘" |
| **수정 요청** | "이 부분이 이상해, 고쳐줘" |
| **데이터 연결** | "@파일명 이 데이터 기반으로 만들어줘" |
