# @파일로 데이터 연결하기

## 지금까지의 한계

지금까지 만든 앱은 AI가 **일반적인 지식**으로 답변합니다.\
실제 GS25 규정이 아니라 AI가 추측한 내용이죠.

> "진짜 우리 회사 규정 데이터를 넣을 수는 없을까?"

**됩니다!** 프로젝트 폴더에 있는 파일을 `@` 기호로 참조하면 됩니다.

{% hint style="info" %}
**MISO 비유**: MISO에서 Knowledge Base를 연결했던 것과 같은 개념입니다.\
다만 여기서는 파일을 직접 프로젝트에 넣고 `@파일명`으로 참조합니다.
{% endhint %}

***

## Step 1: 샘플 데이터 확인하기

프로젝트 폴더에 미리 준비된 규정 데이터 파일이 있습니다.\
좌측 파일 탐색기에서 `data` 폴더를 열어보세요.

| 파일명 | 내용 |
| --- | --- |
| `regulations.md` | 편의점 운영 규정 FAQ 20개 |

파일을 클릭해서 어떤 내용이 들어있는지 한번 살펴보세요.

***

## Step 2: @파일로 데이터 연결하기

Kiro Chat에 아래와 같이 입력합니다.\
`@`를 입력하면 파일 목록이 나타나므로, 거기서 파일을 선택할 수도 있습니다.

{% code title="Kiro Chat에 입력" %}
```
@data/regulations.md 이 파일에 있는 규정 데이터를 기반으로 답변하도록 앱을 수정해줘.

- 사용자가 질문하면 이 데이터에서 가장 관련 있는 규정을 찾아서 답변
- 데이터에 없는 질문이면 "해당 규정은 등록되지 않았습니다. 본사에 문의해주세요"로 안내
- 답변할 때 규정 번호와 카테고리도 함께 표시
```
{% endcode %}

### @파일의 의미

| 표현 | 의미 |
| --- | --- |
| `@data/regulations.md` | "이 파일의 내용을 참고해서" |
| 일반 텍스트 | "이렇게 해줘" (지시사항) |

{% hint style="info" %}
`@`는 AI에게 **"이 파일을 읽어봐"** 라고 알려주는 신호입니다.\
MISO에서 Knowledge Base에 문서를 업로드하는 것과 같은 역할이에요.
{% endhint %}

***

## Step 3: 결과 확인하기

브라우저에서 앱을 새로고침하고, 아까와 같은 질문을 해보세요.

**Before** (@파일 연결 전):
> 질문: "반품 절차가 어떻게 되나요?"\
> 답변: (AI가 일반적인 내용으로 추측한 답변)

**After** (@파일 연결 후):
> 질문: "반품 절차가 어떻게 되나요?"\
> 답변: \[규정 #12 - 반품/교환\] 반품은 입고 후 72시간 이내에... (실제 데이터 기반 답변)

{% hint style="success" %}
**실제 데이터를 기반으로 답변하는 앱이 완성되었습니다!**\
이제 이 앱은 AI의 추측이 아니라, 여러분이 넣은 규정 데이터로 답변합니다.
{% endhint %}

## (참고) 실무에서 활용한다면?

| 상황 | @파일로 넣을 수 있는 것 |
| --- | --- |
| 규정 검색 도우미 | 운영 매뉴얼, 규정집, FAQ 문서 |
| 사고 대응 보고서 | 보고서 양식 템플릿, 과거 사례 |
| 재계약 어시스턴트 | 공헌이익 분석 자료, 협상 가이드 |
| G-ESPA 활동 코치 | 상권 분석 데이터, 성공/실패 사례 |

{% hint style="info" %}
@파일은 여러 개를 동시에 참조할 수도 있습니다.\
예: `@data/regulations.md @data/faq.md 이 두 파일을 참고해서...`
{% endhint %}
