# Steering이란?

## 한마디로

**Steering**은 AI에게 "이 프로젝트에서는 이렇게 해"라고 적어두는 **지침서**입니다.

***

## Steering이 있을 때 vs 없을 때

같은 요청을 했을 때, Steering 유무에 따라 결과가 이렇게 달라집니다:

### Steering 없이 요청한 경우

> 나: "편의점 규정 검색 페이지 만들어줘"

* 디자인이 매번 달라짐
* 영어가 섞여 나올 수 있음
* 답변 톤이 일정하지 않음
* 색상, 폰트 등이 제각각

### Steering 있는 경우

> Steering에 적어둔 것: "GS25 파란색(#0066CC) 사용, 한국어, 존댓말, 모바일 우선"
>
> 나: "편의점 규정 검색 페이지 만들어줘"

* 항상 GS25 파란색이 적용됨
* 모든 텍스트가 한국어
* 답변이 항상 존댓말
* 모바일에서도 잘 보임

{% hint style="success" %}
**핵심 포인트**\
Steering = 한 번 쓰면 매번 반복하지 않아도 되는 **자동 적용 규칙**
{% endhint %}

## MISO에서의 경험과 비교

| MISO | Kiro Steering |
| --- | --- |
| 각 노드의 시스템 프롬프트에 지침 작성 | 프로젝트 전체에 적용되는 지침 한 곳에 작성 |
| 노드마다 따로 설정 | 한 번 작성하면 모든 대화에 자동 적용 |
| 워크플로우 내부에서만 동작 | 코드 생성, 디자인, 답변 등 모든 작업에 적용 |

***

## Steering에는 뭘 적나요?

크게 3가지를 적습니다:

### 1. 프로젝트 소개
> "이 프로젝트는 GS25 점주를 위한 규정 검색 도우미입니다"

### 2. 디자인 규칙
> "GS25 파란색 사용, 모바일 우선, 깔끔한 디자인"

### 3. AI 동작 규칙
> "답변은 존댓말, 3줄 이내, 규정 번호 포함"
