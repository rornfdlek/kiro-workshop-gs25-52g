# 워크샵 정리

## 오늘 한 것

코드를 한 줄도 직접 작성하지 않고, **대화만으로 편의점 업무 도우미 앱을 만들었습니다.**

| Module | 배운 것 | 핵심 한 줄 |
| --- | --- | --- |
| **Module 1** | Steering | AI에게 규칙을 한 번 알려주면 매번 지킨다 |
| **Module 2** | 바이브 코딩 | 한글로 요청하면 AI가 앱을 만들어준다 |
| **Module 3** | Spec | 요구사항을 정리하면 AI가 설계도를 그려준다 |
| **Module 4** | 팀 챌린지 | 배운 것을 조합해서 나만의 앱을 만든다 |

***

## MISO와 Kiro, 뭐가 다른가요?

| | MISO | Kiro |
| --- | --- | --- |
| **만드는 것** | 에이전트 워크플로우 (백엔드 로직) | 사용자가 쓰는 완성된 앱 (UI + 로직) |
| **방법** | 드래그앤드롭으로 노드 연결 | 자연어로 대화 |
| **결과물** | API/워크플로우 | 웹앱, 대시보드 등 |
| **활용** | 비즈니스 로직 자동화 | 현장에서 바로 쓸 수 있는 도구 |

> **ℹ️ 참고**
> **둘은 경쟁이 아니라 보완 관계입니다.**\
> \
> MISO로 **에이전트의 두뇌**(워크플로우)를 만들고,\
> Kiro로 **사용자가 쓰는 앱**(인터페이스)을 만들면\
> 완성된 AI 서비스가 됩니다.

***

## Kiro의 핵심 3가지

다른 AI 도구가 아닌 **Kiro**를 쓰는 이유:

### 1. Steering - 일관된 규칙

> ChatGPT: 매번 "존댓말로, 파란색으로, 한국어로..." 반복해야 함\
> Kiro: **한 번 적으면 끝**. 프로젝트 전체에 자동 적용

### 2. Spec - 체계적 설계

> ChatGPT: 복잡한 앱일수록 앞뒤가 안 맞음\
> Kiro: **요구사항 → 설계 → 구현** 체계적 프로세스

### 3. 프로젝트 컨텍스트

> ChatGPT: 대화가 길어지면 앞에 한 말을 잊음\
> Kiro: **프로젝트 전체 파일을 항상 인식**하고 @파일로 데이터 연결 가능

## 팀 챌린지 결과 발표

각 조의 결과물을 함께 살펴보겠습니다.

발표 순서:
1. 어떤 주제를 선택했는지
2. 앱 데모 (브라우저에서 보여주기)
3. Steering에 어떤 규칙을 넣었는지

**투표**: 발표가 끝나면 "가장 실무에서 쓰고 싶은 앱"에 투표해주세요!
